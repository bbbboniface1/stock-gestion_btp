📋 CAHIER DES CHARGES DE RÉFÉRENCE (à utiliser comme grille de conformité)

[Architecture]
- Frontend React/Next.js, Backend Supabase (PostgreSQL + Auth + API auto), State management Zustand/Redux Toolkit, séparation claire UI / services / DB layer.

[Base de données — STRICT SCHEMA]
- Tables : Users (id, full_name, email unique, role admin|manager|worker, created_at)
- Products (id, name, category indexée, unit, quantity_in_stock, minimum_threshold, location, created_at, updated_at)
- StockMovements (id, product_id FK, type IN|OUT, quantity >0, reason, project_id FK nullable, created_by FK, created_at)
- Projects (id, name, client_name, status active|completed|paused, start_date, end_date)
- ProjectMaterials (id, project_id, product_id, quantity_used)
- Contraintes critiques : quantity_in_stock mis à jour automatiquement (trigger/logique backend), stock négatif INTERDIT.

[Logique métier critique]
- OUT réduit le stock, IN l'augmente, OUT bloqué si quantity > stock disponible, chaque mouvement loggé dans StockMovements.
- RBAC strict : Admin (accès total + suppression définitive), Manager (gère stock/projets, PAS de suppression définitive), Worker (uniquement ajout de mouvements IN/OUT).

[PWA]
- manifest.json valide (icônes 192x192 + 512x512, mode standalone), installabilité réelle testée.
- Mode offline : cache des pages principales, stockage local (IndexedDB/localStorage), queue d'actions offline avec sync automatique au retour réseau.
- Service Worker : cache API responses, stratégie stale-while-revalidate, retry sync en cas d'échec réseau.

[UI / Fonctionnalités]
- Dashboard (stock total, produits critiques < minimum_threshold, mouvements récents, projets actifs)
- Stock (liste, recherche instantanée, filtres catégorie/stock faible, bouton rapide entrée/sortie)
- Mouvements (historique complet, filtres date/produit/projet)
- Projets (création, ajout matériaux utilisés, suivi consommation)

[Authentification]
- Email/password Supabase Auth, session persistante, protection des routes, redirection login automatique si non connecté.

[Qualité logicielle]
- Validation des données frontend ET backend, gestion d'erreurs réseau avec messages clairs, retry automatique sur échec de sync, pagination des listes, lazy loading, requêtes DB optimisées (pas de N+1, index pertinents).

[Seed data]
- 10 produits, 2 projets, 5 utilisateurs, 20 mouvements de stock simulés, réellement présents en base.
